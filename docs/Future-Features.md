# Future Features

DynaTrade currently focuses on a stable dynamic economy runtime for Spigot and Paper servers.

This page describes capabilities planned for future versions. These items are roadmap direction, not separate runtime lines and not guaranteed release commitments.

---

## Available now

The current documented runtime already includes:

- dynamic item pricing driven by player buy and sell activity
- buy and sell command flows
- market GUI browsing and item detail views
- pricing configuration and item-level tuning
- recovery-oriented runtime behavior
- player-aware pressure normalization in the base pricing model
- documented testing and validation evidence

---

## Planned capabilities

Areas currently planned for later evolution include:

- momentum-aware pricing improvements
- additional pricing extensibility for future system features
- analytics and market visibility improvements
- stronger admin diagnostics
- expanded validation and benchmarking tools
- richer configuration examples and operator guidance
- improved operational documentation

---

## Compatibility principles

Future features should preserve:

- the same core pricing engine
- existing configuration compatibility whenever possible
- safe fallback behavior when optional future signals are not available
- recovery and restart behavior
- deterministic pricing calculations
- no unnecessary migration burden for existing servers

---

## What future features are not

Future features should not be presented as:

- a separate runtime line
- a guaranteed delivery promise
- a replacement for the current stable runtime

---

## Roadmap note

Items listed here represent planned system capabilities. They may change based on testing, performance validation, server feedback, and implementation risk.
